# Gen-1-Pokemon-Detector-And-Classifier

## Running the Render Pipeline

The synthetic dataset is generated by running `scripts/render_generation.py`

### Command

```bash
caffeinate -i nohup /Path/To/Blender --background --python /Path/To/render_generation.py > /tmp/render_batch.log 2>&1 &
```

Replace `/Path/To/Blender` with the path to your Blender executable and `/Path/To/render_generation.py` with the absolute path to this repo's `scripts/render_generation.py`.

### Monitoring progress

Open a new terminal and stream the log:

```bash
tail -f /tmp/render_batch.log
```

You'll see `tqdm`'s Pokemon-by-Pokemon progress bar update live. Press `Ctrl+C` to exit the tail — this only stops the `tail` command, not the render itself.

### Checking that it's still running

```bash
ps aux | grep Blender
```

If you see a `Blender --background ...` line, the render is alive.

### Stopping the batch early

```bash
pkill -f "Blender --background"
```

Kills all matching Blender processes. Any in-progress render is discarded; previously completed PNGs stay on disk, so re-running picks up where the kill happened (assuming your script implements skip-if-exists).
